SCP Protocol Unit Tests Configuration

This directory contains Unity unit tests for the Secure Control Protocol (SCP) v1.1 implementation.

Test Structure

  • test_scp_framing.c - COBS framing, CRC validation, message parsing
  • test_scp_crypto.c - HMAC authentication, key management, secure utilities
  • test_scp_session.c - Session management, role authorization, rate limiting
  • test_scp_gateway.c - Gateway integration, message routing, transport handling

Test Categories

Foundational Tests (Phase 2)

  • T014: COBS encoding/decoding validation
  • T015: HMAC authentication verification
  • T016: Session state management validation

Integration Tests

  • Message end-to-end processing
  • Transport interface validation
  • Safety system integration
  • Performance benchmarking

Running Tests

# Run all SCP tests
pio test -f test_scp_*
# Run specific test
pio test -f test_scp_framing
# Run with verbose output
pio test -f test_scp_* -v

Test Requirements

All tests must:

  1. Use Unity framework assertions
  2. Include proper setup/teardown
  3. Test both success and failure cases
  4. Validate error handling and edge cases
  5. Meet timing requirements for real-time constraints
  6. Pass Constitution Check requirements

Coverage Requirements

  • Minimum 90% line coverage for pure logic functions
  • 100% coverage for security-critical functions (auth, validation)
  • Performance tests for timing constraints (<5ms command processing)
  • Memory safety tests (buffer overflows, null pointers)

Test Data

Test vectors and mock data are located in `test_data/` subdirectory:

  • Known HMAC test vectors
  • Valid/invalid frame examples
  • Session state test scenarios
  • Performance benchmark data

Integration with CI/CD

Tests are automatically run on:

  • Pull request validation
  • Pre-commit hooks
  • Release validation
  • Hardware-in-the-loop (HITL) testing

See ../integration/ directory for integration test procedures.
