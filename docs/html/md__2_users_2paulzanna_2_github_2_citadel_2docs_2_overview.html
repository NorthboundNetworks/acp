Citadel Core CLI — Architectural Overview (C Implementation)

Project Code: GCS-CITADEL-01-CLI
System: Scout Ground Control System (Citadel GCS)
Component: Core Command-Line Interface (Citadel Core CLI)
Language: ANSI C (POSIX)
Version: Draft v0.1
Author: [Redacted]
Date: 2025-10-26

📌 Objective

Build a minimal, cross-platform, serial-connected command-line application to:

  • Interface with Scout drone via serial using SCP protocol
  • Display CPU, telemetry, and sensor statistics in real time
  • Operate with minimal dependencies
  • Provide a hardened foundation for future integrations (e.g. Lattice OS, Web API)

🧱 High-Level Architecture

```plaintext +----------------------------—+ | Citadel Core CLI | |----------------------------—| | Serial Port Interface | ← /dev/ttyUSB0, COM3, etc. | SCP Protocol Decoder | | Telemetry Buffer | | Data Display (printf / TUI) | | Optional: Logging | +----------------------------—+

⸻

📦 Module Breakdown

Module Description main.c Entrypoint; argument parsing, init, main loop serial.c / serial.h Low-level serial communication using termios.h scp.c / scp.h Decoder for SCP v1.1 telemetry frames display.c / display.h Formatted terminal output (basic or via ncurses) log.c / log.h (Optional) Audit log for telemetry config.c / config.h Load config from citadel.conf util.c / util.h Timestamps, error handling, memory wrappers

⸻

🛰 Data Flow (Runtime)

[ Scout Drone ] –SCP over Serial--> [ Serial Manager (C) ] │ ↓ [ SCP Frame Decoder (C) ] ↓ [ Telemetry Buffer (structs) ] ↓ [ CLI Display / Optional Logging ]

⸻

📝 Configuration

Basic runtime parameters defined in citadel.conf:

[serial] port = /dev/ttyUSB0 baud = 115200

[display] mode = ascii ; options: ascii, tui

[logging] enabled = true logfile = logs/citadel.log

⸻

📤 Output Modes

Mode Description ASCII (default) Human-readable printf display TUI (optional) Real-time ncurses dashboard Log CSV-style log file for audit/debug

Example ASCII display:

[SCOUT CONNECTED @ 115200 baud]

Telemetry Snapshot (UTC: 14:05:32)

CPU Temp : 48.6 °C CPU Load : 34.2 % Battery : 11.4 V Altitude : 73.2 m Gyro (Z) : 0.042 rad/s GPS Fix : 3D (9 sats)

🔒 Security & Reliability • Optional CRC validation for SCP packets • Graceful reconnection on serial drop • Log rotation support • Static binary build (for hardened deployment) • Built-in timeout watchdog on data freshness

⸻

🛠 Build & Platform Support • Build system: Makefile (cross-platform) • Dependencies: Standard C, termios.h, optional ncurses • Targets: Linux (primary), macOS, Windows via MinGW or WSL • Output: citadel_cli executable

🔚 Summary

The Citadel Core CLI provides a lean, serial-driven, zero-dependency command-line interface to Scout drones. It forms the operational base for future system integration (e.g. Lattice OS, Web UI, or Citadel-Pivot extensions), while remaining robust, portable, and easily audited.
