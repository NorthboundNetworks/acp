Summary

Build a portable ANSI C (C11) CLI that connects to a serial device (default 115200 baud), decodes SCP v1.1 telemetry frames, and renders a stable ASCII display by default (optional ncurses TUI via flag). Provide bounded disconnect recovery (exponential backoff up to 5 minutes then non‑zero exit), and optional CSV logging with UTC ISO 8601 timestamps. Target Linux and macOS with a Makefile build, zero warnings, hardening flags, and basic unit tests for the decoder.

Technical Context

Language/Version: C11 (ANSI C)
Primary Dependencies: POSIX termios (serial I/O), optional ncurses (TUI), standard C library
Storage: N/A (in‑memory only; optional CSV file logging)
Testing: Unity (single‑file C unit tests) for decoder and utility functions
Target Platform: Linux and macOS (POSIX); Windows deferred (WSL/MinGW later)
Project Type: Single CLI project (Makefile)
Performance Goals: p95 refresh ≤ 200 ms at 115200 baud; no frame drops
Constraints: ≤ 50 MB RSS; startup ≤ 2 s; ASCII default; bounded reconnect ≤ 5 minutes
Scale/Scope: MVP decoder + display + logging; TUI opt‑in and minimal

Makefile Targets:

  • all: Build main binary with release flags
  • debug: Build with debug symbols and no optimization
  • test: Compile and run unit tests with coverage
  • lint: Run clang-tidy and cppcheck static analysis
  • clean: Remove build artifacts
  • install: Install binary to system path
  • valgrind: Run tests under memory checker

Build Configuration:

  • Release: gcc -std=c11 -O2 -g -Wall -Wextra -Werror -fstack-protector-strong -D_FORTIFY_SOURCE=2
  • Debug: gcc -std=c11 -O0 -g3 -Wall -Wextra -Werror -fsanitize=address
  • Linker: -Wl,-z,relro,-z,now (Linux), -Wl,-bind_at_load (macOS)
  • ncurses: Conditional linking via pkg-config detection
