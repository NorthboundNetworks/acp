Environment Variables (Eunice Platform)

Authoritative list of required vs optional environment variables for local development and deployment. (Remediates problems.md item 45.)

Required (Startup Will Fail if Missing)

  • GOOGLE_API_KEY: Google Custom Search API key used by network-service for web research.
  • GOOGLE_SEARCH_ENGINE_ID: Google Custom Search Engine ID (CX) paired with above key.
  • AUTH_SECRET_KEY: Secret key for signing JWT tokens in auth-service.

Recommended / Optional

  • CORE_API_KEY: (Optional) CORE API key for academic literature sources.
  • OPENALEX_EMAIL: (Optional) Contact email for OpenAlex polite usage identification.
  • OPENAI_API_KEY: OpenAI models invocation via network-service (if provider set to openai).
  • ANTHROPIC_API_KEY: Anthropic models invocation via network-service (if provider set to anthropic).
  • XAI_API_KEY: xAI (Grok) models invocation via network-service (if provider set to xai).
  • NCBI_API_KEY: NCBI E-utilities rate limit expansion for literature queries.

Service URL / Ports

  • EUNICE_HOST: Host binding for API gateway (default 127.0.0.1 in dev).
  • EUNICE_WEB_PORT: External web port for API gateway.
  • EUNICE_MCP_PORT: Port for internal MCP endpoints (if exposed).
  • DATABASE_URL: PostgreSQL connection string (services fall back to in-container defaults if unset).
  • DATABASE_SERVICE_URL: Internal HTTP URL for database-service MCP endpoint (research-manager uses). Prefer this over deprecated DATABASE_MCP_URL.
  • NETWORK_SERVICE_URL: Internal HTTP URL for network-service (research-manager LLM calls).

Research Manager

  • FORCE_PLANBRIEF_V1 (default true): Forces override to PlanBrief v1 schema if legacy schema detected.
  • PLANBRIEF_V1_SCHEMA: Override path to plan_brief_v1.schema.json.

Logging / Telemetry

  • LOG_LEVEL: Per-service log level (INFO/DEBUG/etc.).
  • METRICS_PORT: (Internal) Port for Prometheus metrics exposure (should not be host-exposed in production for internal services).

Development Flags

  • DEVELOPMENT_MODE: Enables watchfiles/hot-reload logic inside container start scripts.
  • WATCHFILES_FORCE_POLLING: Set to 1 when host file system events are unreliable (e.g. some Docker Desktop setups).

Security Notes

Do NOT commit real API keys. The root .gitignore excludes .env. Provide sample values in deployment templates only. Update this file when adding new environment-controlled features.
