Preflight Checklist Contract

Created: 2025-10-25
Branch: 001-scout-firmware
Spec: ../spec.md

Purpose

Define the minimal preflight checklist used to gate arming, with inputs, thresholds, pass/fail rules, and operator feedback.

Checklist Items

  • Step: Mode check
    • Input: current mode
    • Pass: mode == FLIGHT
    • Fail: CONFIG mode active
    • Operator feedback: "Arming blocked: CONFIG mode active"
  • Step: Receiver link
    • Input: link quality (0–100)
    • Pass: quality >= MIN_LINK_QUALITY (e.g., 50)
    • Fail: below threshold or missing
    • Operator feedback: "Arming blocked: receiver link too weak"
  • Step: Sensor health — IMU/MAG/BARO
    • Input: health flags per sensor
    • Pass: all required sensors OK
    • Fail: any required sensor FAULT
    • Operator feedback: "Arming blocked: sensor fault (IMU/MAG/BARO)"
  • Step: CPU budget
    • Input: total CPU utilization (%)
    • Pass: total <= 70%
    • Fail: > 70%
    • Operator feedback: "Arming blocked: CPU budget exceeded"
  • Step: Vehicle‑launch envelope (if moving)
    • Inputs: speed (km/h), attitude (roll/pitch), sensor OK, link OK, CPU OK
    • Pass: speed <= 80 km/h AND |roll|,|pitch| <= 15° AND other gates pass
    • Fail: any condition violated
    • Operator feedback: "Arming blocked: vehicle‑launch envelope not satisfied"

Evidence & Integration

  • CLI: status shows safety summary and last preflight failure reason
  • Telemetry: STATUSTEXT includes preflight failure message on rejection
  • Tests: see tasks T026 (evaluation) and T033 (bench session with arming attempts)
