Data Model: Custom Firmware for Scout ISR Drone

Created: 2025-10-25 Branch: 001-scout-firmware Spec: ../spec.md

Entities

Configuration Profile

  • Fields: parameters (key→value), version metadata, checksum
  • Rules: round‑trip export/import equivalence; validation on set; persist on save

Safety State

  • Fields: sensor_health (IMU, MAG, BARO, GPS), receiver_link, cpu_budget_ok, mode, arming_allowed
  • Rules: arming_allowed derives from gates; any gate fail → disarm/deny arm

Flight Mode

  • Fields: mode_id (hover, loiter, forward), constraints, transitions
  • Rules: transition only when constraints satisfied; heading‑reliant modes require MAG OK

Telemetry Packet

  • Fields: heartbeat, system_status, pose (att/alt), battery, link_quality
  • Rules: emit ≥1 Hz baseline; reflect safety state; indicate NO FIX when GPS unavailable

Preflight Checklist

  • Fields: steps (ordered), result per step, last_run_ts
  • Rules: checklist must pass before arming; results surfaced to operator

Sensor Health Report

  • Fields: per‑sensor state (OK/FAULT/CAL_REQ), calibration status, last_fault
  • Rules: health affects checklist, safety state, and mode availability
