Quickstart: Custom Firmware for Scout ISR Drone

Created: 2025-10-25 Branch: 001-scout-firmware Spec: ../spec.md

Prerequisites

Hardware Requirements

  • ESP32-S3 development board (freenove_esp32_s3_wroom)
  • USB-C cable for programming and CLI access
  • Scout sensor suite: MPU-6050 (IMU), HMC5883L (compass), BMP388 (barometer), NEO-6M (GPS)
  • Breadboard or PCB for sensor connections (see wiring.md)
  • CRITICAL: Propellers removed for all bench testing

Software Requirements

  • macOS with PlatformIO Core installed
  • Python 3.8+ with virtual environment support
  • Serial terminal (screen, minicom, or PlatformIO device monitor)

Environment Setup

CRITICAL: Always activate the Python virtual environment before any firmware operations:

cd /path/to/Scout
source .venv/bin/activate

For new terminal sessions: The virtual environment must be activated each time. Options:

  1. Setup script: ./setup-dev.sh (includes environment activation)
  2. direnv (recommended): brew install direnv && direnv allow (auto-activation)
  3. Manual: source .venv/bin/activate in each terminal

Verification: Check that (Scout) appears in your prompt when environment is active.

Build & Flash Procedure

Step-by-step Build Process

  1. Environment Verification:

    # Ensure virtual environment is active
    source .venv/bin/activate
    # Verify PlatformIO is available
    pio --version
  2. Hardware Connection:
    • Connect ESP32-S3 board via USB-C
    • Verify device recognition: ls /dev/cu.usbmodem* (macOS)
    • Put board in download mode if needed (hold BOOT, press RESET)

  3. Project Build:

    # Clean build (recommended for first build)
    pio run -e freenove_esp32_s3_wroom --target clean
    # Build firmware
    pio run -e freenove_esp32_s3_wroom

  4. Flash Firmware:

    # Upload to device
    pio run -e freenove_esp32_s3_wroom --target upload
    # Monitor serial output
    pio device monitor -p /dev/cu.usbmodem*

Common Build Issues & Solutions

Issue Cause Solution
pio: command not found PlatformIO not in PATH Install PlatformIO Core: pip install platformio
Device not found USB not connected Check USB cable, try different port
Permission denied User lacks device access Add user to dialout group or use sudo
Build errors Missing dependencies Run pio lib install to fetch libraries
Upload timeout Board not in download mode Hold BOOT button while uploading

Basic Verification

  1. Connect serial console, run version, status, stats.
  2. Confirm CPU utilization ≤70% and loop rates within ±5% of configured values.
  3. Use get/set, then save and reboot; verify dump reflects persisted changes.
  4. Exercise sensor checks: verify IMU/MAG/BARO/GPS health in status.
  5. Attempt arming in configuration (WiFi/AP) mode (should be blocked) and in flight mode with all safety gates passing (should succeed).
  6. Verify baseline telemetry: periodic status and arm/disarm round‑trip.

Baseline Profile (for acceptance)

  • Board: freenove_esp32_s3_wroom
  • Mode default: CONFIG after boot; FLIGHT when explicitly set via CLI
  • Loop rates: control=200 Hz, telemetry=1–2 Hz (target), health=10 Hz
  • CPU budget: ≤70% total during bench tests
  • Telemetry: HEARTBEAT/SYS_STATUS ≥1 Hz; jitter ≤±250 ms; ≥99% delivery over 10 minutes
  • WiFi/AP: disabled in FLIGHT; no network CLI at any time

To switch modes for tests:

  • Enter FLIGHT: set mode FLIGHT → save → reboot, or via dedicated CLI if present
  • Return to CONFIG: set mode CONFIG (arming must be blocked in CONFIG)

Safety Checklist (excerpt)

  • Propellers removed for bench tests
  • Configuration mode active for CLI operations
  • Verify failsafe behavior on link loss (expect disarm ≤1s)
