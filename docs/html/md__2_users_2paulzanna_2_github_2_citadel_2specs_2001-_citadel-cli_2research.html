Research — Citadel Core CLI (v0.1)

Date: 2025-10-26
Branch: 001-Citadel-cli
Spec: specs/001-Citadel-cli/spec.md

Decisions

  1. Unit Test Framework: Unity (single-file C tests)
  • Rationale: Minimal overhead, ANSI C compatible, easy to vendor, good fit for embedded/CLI.
  • Alternatives: cmocka (POSIX, requires lib), Criterion (richer but heavier). Unity chosen for simplicity.
  1. SCP Mock Frame Schema
  • Decision: Include CPU load %, CPU temp C, battery V, gyro_z rad/s, GPS fix enum, timestamp_utc.
  • Rationale: Matches MVP display and CSV columns; sufficient for decoder validation.
  • Alternatives: Full TLV or binary payload expansion deferred to v0.2.
  1. Backoff Policy for Disconnects
  • Decision: Exponential backoff up to 5 minutes, then exit non-zero.
  • Rationale: Balances resiliency with predictability for operators/scripts.
  • Alternatives: Infinite retry (too opaque), immediate exit (too brittle).
  1. Logging Format
  • Decision: CSV with header row and fixed columns (timestamp_utc, cpu_load_pct, cpu_temp_c, battery_v, imu_gyro_z, gps_fix).
  • Rationale: Tool-friendly, diffable, low overhead; aligns with auditability.
  • Alternatives: JSONL (verbose), raw SCP (opaque without decoder).
  1. Timestamps
  • Decision: UTC ISO 8601 with Z.
  • Rationale: Unambiguous, audit-friendly, standard in gov/mil contexts.
  1. Build Toolchain & Flags
  • Decision: Makefile with gcc/clang; -O2 -g -Wall -Wextra -Werror; hardening: -fstack-protector-strong -D_FORTIFY_SOURCE=2; link: -Wl,-z,relro,-z,now (Linux).
  • Rationale: Constitution quality/security gates; portable and auditable.

Alternatives Considered (Summary)

  • Test frameworks: Unity vs cmocka vs Criterion — chose Unity for minimalism.
  • Logging: CSV vs JSONL vs raw — chose CSV.
  • Retry policy: infinite vs bounded vs none — chose bounded.
  • TUI: default vs opt-in — opted for opt-in to keep ASCII baseline reliable.
