Degrade Policy Contract

Created: 2025-10-25
Branch: 001-scout-firmware
Spec: ../spec.md

Purpose

Define safe degrade behaviors per fault type so the system restricts modes, increases stability margins, or disarms if safety is compromised. Includes operator feedback and telemetry signaling.

General Rules

  • Safety first: if a fault compromises safe control while armed, disarm.
  • Prefer graceful degrade of non-critical features before disarm.
  • Always surface health faults in status and via STATUSTEXT.
  • Arming gate consumes health state; if any REQUIRED sensor is unhealthy, arming_allowed=false.

Fault Classes and Actions

IMU Fault (FR-014)

  • Pre-arm: block arming.
  • In flight: immediate disarm (critical for stabilization); STATUSTEXT="IMU fault — disarm".

Magnetometer Fault (FR-015)

  • Pre-arm: heading-reliant modes unavailable; arming allowed for attitude-only modes if other gates pass.
  • In flight: exit heading-reliant modes to attitude-stabilized mode; reject new heading-reliant mode requests; STATUSTEXT with reason.

Barometer Fault or Drift Out of Bounds (FR-016)

  • Pre-arm: do not block arming if other gates pass; ignore transient spikes.
  • In flight: disable baro-derived altitude hold; maintain manual/attitude modes; STATUSTEXT with reason.

GPS No Fix / Poor Quality (FR-017)

  • Pre-arm: do not require GPS unless configured; GPS-dependent features unavailable.
  • In flight: disable GPS-dependent behaviors; maintain non-GPS modes; STATUSTEXT with reason.

Receiver Link Loss (FR-002)

  • In flight: failsafe triggers disarm ≤1 s from loss detection; STATUSTEXT and COMMAND_ACK on recovery.

CPU Budget Exceeded (FR-005, SC-004)

  • Pre-arm: set arming_allowed=false when total CPU > 70% sustained over sampling window.
  • In flight: degrade non-critical features (reduce telemetry rate to 1 Hz floor; disable optional payload processing), then refuse arming until CPU ≤70%.
  • STATUSTEXT with "CPU budget exceeded" and indication of degraded features.

Operator Feedback

  • CLI status lists arming_allowed, cpu_budget_ok, per-sensor health, and last fault reason.
  • STATUSTEXT events for: preflight gate rejections, mode unavailability, degrade activations, disarm on critical faults.

Configuration Hooks

  • Parameters control thresholds and required sensors (e.g., REQUIRED_SENSORS, CPU_MAX_PCT, BARO_DRIFT_MAX).
  • Defaults bias toward safety; optional features disabled by default until validated.
