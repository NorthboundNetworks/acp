Required Signals

  • CLI status: mode, armed, arming_allowed, cpu_budget_ok, sensor health (IMU/MAG/BARO/GPS), battery summary, link quality.
  • CLI stats: per-loop target/measured rate, p95 loop time, total CPU.
  • Telemetry: HEARTBEAT, SYS_STATUS, STATUSTEXT at prescribed rates.
  • Optional: blackbox logs for timing and sensor traces (if enabled).

Troubleshooting Checklist

  • Arming failures: check status safety fields and STATUSTEXT reason.
  • Timing issues: review stats p95 and CPU total over 10-minute bench.
  • Sensor anomalies: verify health flags and calibration states; re-run calibrations as needed.
  • Link problems: confirm COMMAND_ACK results and loss-of-link disarm behavior.

Evidence for PRs

  • Attach status/stats snapshots before and after changes.
  • Provide telemetry logs for 10-minute bench session.
  • Include golden CLI transcripts for modified commands.
