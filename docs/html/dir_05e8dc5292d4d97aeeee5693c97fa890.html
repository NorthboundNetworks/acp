Detailed Description

News Collection System

This module provides a Python implementation of the news collection and analysis system, equivalent to the TypeScript DataCollector.

Features

  • News Collection: Automatically collects news articles from various sources using NewsAPI
  • Country Detection: Identifies countries mentioned in news articles
  • Sentiment Analysis: Basic sentiment analysis of news content
  • Categorization: Automatically categorizes news into conflict, political, economic, social, and security categories
  • Alert Generation: Creates alerts for significant events based on sentiment and content analysis
  • Stability Metrics: Calculates stability scores for countries based on news analysis
  • Scheduling: Automated collection on configurable schedules

Installation

  1. Install required dependencies:
    pip install -r requirements.txt
  2. Set up environment variables:
    cp .env.example .env
    # Edit .env with your API keys
  3. Initialize the database:
    from database import Base, engine
    Base.metadata.create_all(bind=engine)

Usage

Basic Usage

import asyncio
from sqlalchemy.orm import Session
from newsCollector import DataCollector
from database import SessionLocal
async def main():
db_session = SessionLocal()
collector = DataCollector(database_session=db_session)
# Start automated collection
await collector.start_collection()
# Keep running
while True:
await asyncio.sleep(60)
if __name__ == "__main__":
asyncio.run(main())

Configuration

Environment Variables

  • NEWS_API_KEY: Your NewsAPI key for article collection
  • REST_COUNTRIES_API: REST Countries API endpoint
  • WORLD_BANK_API_URL: World Bank API endpoint
  • ALPHA_VANTAGE_API_KEY: Alpha Vantage API key for financial data
  • NEWS_COLLECTION_SCHEDULE: Cron schedule for news collection (default: every 30 minutes)
  • STABILITY_UPDATE_SCHEDULE: Cron schedule for stability updates (default: every 6 hours)

Database Models

The system uses the following database models:

  • Country: Store country information and peace index data
  • NewsArticle: Store collected news articles with sentiment and relevance scores
  • Alert: Store generated alerts for significant events
  • StabilityMetric: Store calculated stability metrics for countries

API Integration

NewsAPI

The system integrates with NewsAPI to collect real-time news articles. You need to:

  1. Sign up at https://newsapi.org/
  2. Get your API key
  3. Add it to your .env file

REST Countries API

Used to initialize country data and mapping. No API key required.

World Bank API

For economic indicators (optional). May require API key for higher rate limits.

Sentiment Analysis

The current implementation includes a basic keyword-based sentiment analysis. For production use, consider integrating with:

  • TextBlob
  • VADER Sentiment
  • spaCy with sentiment models
  • Transformers-based models

Alert System

Alerts are generated based on:

  • Negative sentiment scores (< -0.3)
  • Conflict or security-related categories
  • High relevance scores (> 0.5)

Alert severities:

  • CRITICAL: Contains urgent/critical keywords
  • HIGH: Contains serious/significant keywords
  • MEDIUM: Contains concern/tension keywords
  • LOW: Default level

Stability Metrics

The system calculates four stability metrics:

  1. Political Stability: Based on sentiment and political news
  2. Economic Stability: Based on economic news sentiment
  3. Social Cohesion: Based on social news and sentiment
  4. Security Index: Based on conflict and security news

Overall score is a weighted average of these metrics.

Peace Index Integration

The system includes 2024 Global Peace Index scores for major countries, used as baseline stability indicators.

WebSocket Integration

If a WebSocket service is provided, the system can broadcast:

  • New alerts
  • Stability metric updates
  • Real-time news updates

Error Handling

The system includes comprehensive error handling and logging. All errors are logged with appropriate context for debugging.

Production Considerations

For production deployment:

  1. Database: Consider using PostgreSQL instead of SQLite
  2. Caching: Implement Redis for caching API responses
  3. Rate Limiting: Implement proper rate limiting for API calls
  4. Monitoring: Add monitoring and alerting for the collection process
  5. Scaling: Consider using Celery for distributed task processing
  6. Security: Implement proper API key management and rotation
  7. NLP: Upgrade to advanced NLP libraries for better accuracy

Troubleshooting

Common Issues

  1. API Rate Limits: If you hit rate limits, increase delays between requests
  2. Database Locks: If using SQLite in production, consider PostgreSQL
  3. Memory Usage: For large-scale collection, implement data archiving
  4. Missing Dependencies: Ensure all requirements are installed
