SCP client smoke test

Quick host-side smoke test for the Secure Control Protocol (SCP) v1.1 to validate basic command/telemetry flows over a serial link.

Prerequisites

  • Firmware built and flashed for your board (PlatformIO env freenove_esp32_s3_wroom)
  • USB serial connected to the ESP32-S3
  • Python 3 with the project's virtualenv active (VS Code will do this automatically)
  • pyserial installed (the repo tasks/tools will install it on first use)

Run

  • Send ARM and listen for responses and status:
    • ./tools/scp_client_smoke.py –port /dev/tty.usbserial-XXXX –arm –listen 5
  • Set telemetry rate to 5 Hz and listen for status:
    • ./tools/scp_client_smoke.py –port /dev/tty.usbserial-XXXX –set-rate 5 –listen 10
  • DISARM and observe audit/ack:
    • ./tools/scp_client_smoke.py –port /dev/tty.usbserial-XXXX –disarm –listen 5

Notes:

  • Omit –port to auto-detect the first USB serial device on macOS.
  • Default baud is 115200; adjust with –baud if you changed it in firmware.

Expected output

  • ACK or ERR responses for commands, e.g.:
    • ACK seq=2 acked=1 session=1 result=0x00 time=3ms ts=...
    • ERR seq=4 err=0x03 failed_seq=3 session=1 (SCP_ERR_SAFETY_REJECT when arming blocked)
  • TLM_STATUS lines at the configured rate with battery, mode, GPS and pose summary.
  • TLM_AUDIT lines for security/audit events if enabled.

Troubleshooting

  • No output: check the serial port path and that the device is running with SCP gateway started.
  • Framing errors: ensure you are not running another monitor on the same port.
  • Unexpected ERR 0x04 (AUTH_FAIL): device may require session/auth; use serial CLI to relax/enable SCP settings as needed.
