Eunice v0.5.8 Architecture — Evidence Graph Hardening

Date: 12-08-2025
Owner: Paul Zanna
Status: Implementation-Ready

1) Executive Summary

v0.5.8 hardens the literature pipeline to produce deterministic, citation-faithful outputs. It introduces a canonicalised Evidence Graph, page/section anchors for citations, near-duplicate collapse, externalised prompts/config, and a deterministic cache. The result is repeatable runs with high-confidence references and traceable provenance.

2) Scope

Goals

  • Deterministic ingestion from PubMed, arXiv, Semantic Scholar, CORE, Crossref (via MCP).
  • Canonical IDs (DOI/PMID/arXiv/URL hash) with duplicate collapse.
  • PDF/HTML anchor extraction (page/section/figure/caption; text quotes).
  • Evidence Graph persisted in DB with claim↔source edges (supports/contradicts/cites).
  • Externalised prompts/templates; prompt hashing; profile-based selection (UG/MSc/PhD).
  • Citation-weaving in generated reports with source_id#page anchors.
  • Caching & seeds for reproducible runs.

Non‑Goals (now)

  • PRISMA triage UI, human labelling loops.
  • Protocol/BOM generation (v0.6.1).
  • Full-blown web UI (minimal CLI/TUI only).

Key properties

  • Determinism: All LLM calls seeded; cache keys include prompt-hash & model-id.
  • Idempotence: Safe re-runs; content-addressed storage for fetched artefacts.
  • Provenance: Every claim linked to specific source anchors (page/section/figure).

4) Components

4.1 Orchestrator (CLI/TUI)

  • Runs topic jobs; emits run manifests (inputs, prompt-hashes, model-id, tool versions).
  • Exposes commands: search, ingest, dedup, anchor, evidence-graph, render, full-run.

4.2 MCP Connectors

  • Tools: search(query), get_metadata(id), get_fulltext(id|url).
  • Uniform response schema; rate-limited with backoff; response hashing for cache.

4.3 Normaliser

  • Canonical field mapping: title, authors[], year, venue, ids{{doi,pmid,arxiv}}, urls[], abstract.
  • Title/author normalisation (case, punctuation, unicode NFC).

4.4 Canonicaliser/Deduplicator

  • Priority: DOI → arXiv → PMID → URL hash.
  • Fuzzy match (title trigram Jaccard; author-set overlap; ±1 year tolerance).
  • Thresholds: t_title ≥ 0.88, t_authors ≥ 0.6. Tunable in config.
  • Produces source (canonical) and source_variant rows; merges metadata.

4.5 Fetch + Parse

  • PDF/HTML download with checksum + content-type.
  • PDF text per page; capture headings hierarchy; figure/table captions; references section.

4.6 Anchor Extractor

  • Anchors: page, section_path, figure_id?, caption_text?, quote{start,end,text}.
  • Quote-matching with fuzzy windowing; confidence score; low-confidence flagged for re-run.

4.7 Evidence Graph Builder

  • Nodes: claim (hash(text)), source.
  • Edges: supports|contradicts|cites with anchor_id and optional rationale snippet.
  • Stores per-claim attribution set used by renderer & evaluation harness.

4.8 Report Generator + Citation Weaver

  • Assembles narrative per profile (UG/MSc/PhD later), inserts superscripts mapped to source_id#page.
  • Broken-link detector reports missing or low-confidence anchors.

4.9 Prompt Loader (externalised)

  • Loads prompts/*.json (schema-validated). Hash = SHA256 of canonical JSON (whitespace-insensitive).

4.10 Deterministic Cache

  • Key: (stage, normalized-input-hash, prompt-hash, model-id, tool-version).
  • TTL & invalidation by prompt or connector version change.

4.11 Telemetry & Run Manifests

  • JSONL logs; latencies; cache hits; connector call counts.
  • Run manifest: inputs, hashes, versions, seeds; saved alongside outputs.

7) Algorithms & Heuristics

7.1 Canonicalisation & Dedup

  1. Prefer DOI; else arXiv; else PMID; else URL content-hash.
  2. Title normalise: lowercase, strip punctuation/stopwords, Unicode NFKD→NFC.
  3. Compute trigram Jaccard; author overlap (surname+initials); year ±1.
  4. Match if: Jaccard ≥ t_title AND (author_overlap ≥ t_authors OR year_close).
  5. Collapse variants; persist strongest metadata.

7.2 Anchor Extraction

  • Page-aware search for quotes; window ±N chars; fallback to fuzzy (token-level Levenshtein).
  • Assign confidence; <0.7 flagged for re-run/manual check list.

7.3 Determinism

  • Seed all stochastic ops; record seeds in run manifest.
  • Cache before LLM calls; only miss when hash set changes.

9) Observability

  • JSONL logs per stage; Metrics: p50/p95 latency, cache hit ratio, anchor-confidence histogram.
  • Artefacts directory: raw fetches, parsed text, anchors.json, evidence_graph.json, report.md, run_manifest.json.

10) Security & Licensing

  • macOS sandbox profiles for MCP connectors (net/file limitations).
  • Secret handling: env vars → in-memory only; no logs.
  • PDF licence note field; blocklist of publishers if required.

11) Performance Targets

  • Single-topic (100–200 candidates): end-to-end ≤ 10 min on laptop; cache-hit rerun ≤ 90 s.
  • Dedup collapse rate ≥ 90% on curated dup set.
  • Anchor accuracy ≥ 95% on sampled checks.

12) Testing & CI

  • Golden topics with expected citations & page anchors.
  • Property tests for title/author perturbations.
  • Anchor QA: sample K anchors, compute accuracy; fail <95%.
  • Determinism test: same inputs → identical outputs & manifests.

13) Migration Plan

  1. Add new tables via migration; backfill canonical IDs for existing records.
  2. Move embedded prompts to /prompts/*.json; compute & store SHA256.
  3. Introduce cache; run A/B on a small topic to validate determinism.
  4. Flip default to external prompts after CI passes.

14) Risks & Mitigations

  • Rate limits: per-host backoff + caching.
  • Anchor drift across PDF versions: use content hashes; re-anchor when hash changes.
  • Model drift: pin model-id; prompt-hash gating in cache.
  • Licence issues: metadata-first; fetch PDF only when allowed; record licence flags.

15) Acceptance (DoD)

  • Repeat run → identical report & citations.
  • ≥95% correct page anchors on sample set.
  • ≥90% duplicate collapse on curated dup set.
  • Zero embedded prompts; all in /prompts/*.json with schema validation.
  • CI publishes run manifests & artefacts for two golden topics.

16) Developer Notes

  • Prefer pure functions and explicit inputs/outputs per stage.
  • Use content-addressed storage (hash-named files) under .eunice/artifacts/.
  • Export a single evidence_graph.json for downstream v0.5.9+.
