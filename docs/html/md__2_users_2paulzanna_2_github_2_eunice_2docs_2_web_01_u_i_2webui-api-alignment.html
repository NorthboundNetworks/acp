Web UI ↔ API Alignment (Post-Approval Flow)

Date: 2025-08-13 Base URL: http://localhost:8000

This document maps Eunice Web UI needs to the actual API Gateway endpoints discovered from /openapi.json.

Core resources

  • Projects
    • List: GET /projects
    • Get: GET /projects/{project_id}
    • Create: POST /projects
    • Update: PUT /projects/{project_id}
    • Delete: DELETE /projects/{project_id}
    • Stats: GET /projects/{project_id}/stats
    • Hierarchy: GET /projects/{project_id}/hierarchy
  • Topics
    • List for project: GET /projects/{project_id}/topics
    • Get: GET /topics/{topic_id}
    • Create: POST /projects/{project_id}/topics
    • Update: PUT /topics/{topic_id}
    • Delete: DELETE /topics/{topic_id}
    • Stats: GET /topics/{topic_id}/stats
    • AI Plan generation (structured): POST /topics/{topic_id}/ai-plans
    • AI Plan enqueue (RM inbox): POST /topics/{topic_id}/ai-plans:enqueue
  • Plans
    • List for topic: GET /topics/{topic_id}/plans
    • Get: GET /plans/{plan_id}
    • Update: PUT /plans/{plan_id}
    • Delete: DELETE /plans/{plan_id}
    • Approve: POST /plans/{plan_id}/approve
    • Stats: GET /plans/{plan_id}/stats (NOT IMPLEMENTED in current gateway; progress/task counts are embedded in plan response)
  • Tasks
    • Create: POST /plans/{plan_id}/tasks
    • List for plan: GET /plans/{plan_id}/tasks
    • Get: GET /tasks/{task_id}
    • Update: PUT /tasks/{task_id}
    • Delete: DELETE /tasks/{task_id}

Literature Extraction

  • Start: POST /literature/extraction/start
  • Get record: GET /literature/extraction/{record_id}
  • Status: GET /literature/extraction/status
  • Legacy redirects also available under /literature/*

Protocols

  • Generate from synthesis blocks: POST /protocol/generate
  • Price BOM: POST /protocol/bom/price
  • Get playbook: GET /protocol/playbook/{playbook_id}

Post-Approval UI mapping

  • Approve Plan button → POST /plans/{plan_id}/approve
  • Kickoff Wizard submit → create one or more Tasks via POST /plans/{plan_id}/tasks
    • literature_run → task_type: "literature_review"; metadata.kind: "literature_run"; metadata.payload: { literature: {...} }
    • experiment_design → task_type: "analysis"; metadata.kind: "experiment_design"; metadata.payload: { experiment: {...} }
    • materials_build → task_type: "analysis"; metadata.kind: "materials_build"; metadata.payload: { materials: {...} }
  • Progress view → poll GET /plans/{plan_id}/tasks every 5s
  • Plan details tabs → GET /plans/{plan_id} and GET /plans/{plan_id}/stats
  • Deliverables
    • Protocol preview (if metadata.playbook_id) → GET /protocol/playbook/{playbook_id}
    • BOM pricing (optional) → POST /protocol/bom/price

Notes

  • No /kickoff or /stream/jobs endpoints exist; the UI uses task creation + polling instead of SSE.
  • Use VITE_API_BASE_URL to point the frontend to the API Gateway.
  • Plan stats endpoint missing: adapt UI to derived data already present (tasks_count, completed_tasks, progress).
