Country Data Centralization - Completion Summary

Overview

All country listings, peace index data, and related metadata have been fully migrated from hardcoded implementations and configuration files to the SQLite database (data/sql_app.db). The database is now the single source of truth for country, code, and peace index data, as described in docs/DATABASE_SCHEMA_DOCUMENTATION.md.

Changes Made

1. Database-Centric Data Storage

  • Location: data/sql_app.db (see docs/DATABASE_SCHEMA_DOCUMENTATION.md)
  • Tables:
    • countries: Master list of 250 countries, demographics, peace index
    • country_status: Status values for mapping and analysis
  • Configuration: config/countries.json now only used for auxiliary metadata, not as a primary data source

2. Configuration Access Functions

  • Location: config/config.py
  • Functions:
    • get_countries_list(): Loads country list from database
    • get_country_name_mapping(): Loads name mappings from database
    • Peace index and other analytics now accessed via database queries

3. Refactoring of Data Collectors

  • Files: backend/geopolitics/newsCollector.py, financeCollector.py, geopolAnalytics.py
  • Changes:
    • All country and peace index data loaded from the database
    • Removed all hardcoded and config-based data loading
    • Initialization now queries the database directly

4. Test Updates

  • File: tests/test_newsCollector.py
  • Changes:
    • Tests now use database fixtures for country and peace index data
    • No references to config-based data remain

Benefits Achieved

✅ Centralization

  • All country and peace index data in a single, authoritative database
  • Easy to maintain and update via database scripts
  • Consistent data across the entire application

✅ Maintainability

  • No more hardcoded or config-based data scattered across files
  • Single point of truth for all country-related information
