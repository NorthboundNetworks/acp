Audit Message Format Validation

TLM_AUDIT Message Structure

  • Verify TLM_AUDIT messages contain all required fields per contract
  • Validate event_type field correctly categorizes security events (1-4)
  • Confirm source_session field accurately identifies originating session
  • Check operator_role field matches session role (1=Commander, 2=Pilot, 3=Observer)
  • Verify result_code field reflects actual command processing outcome
  • Validate timestamp_ms field accuracy within ±10ms of actual event time
  • Confirm details field contains relevant security context (16 bytes)
  • Test message size compliance with SCP_MAX_PAYLOAD_SIZE limit

Audit Event Generation Timing

  • Verify audit events generated within 50ms of security violation detection
  • Confirm command audit events sent before command ACK/ERR responses
  • Validate session audit events triggered during session lifecycle changes
  • Test audit event ordering matches chronological event sequence

Audit Message Transmission

  • Confirm TLM_AUDIT messages sent to all authorized operators
  • Verify audit messages use appropriate priority level (HIGH for security events)
  • Test audit transmission doesn't block command processing pipeline
  • Validate audit message delivery even during high system load
