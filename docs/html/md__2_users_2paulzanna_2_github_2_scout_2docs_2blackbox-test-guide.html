Blackbox Integration Test Guide

This guide provides manual test steps to validate end-to-end blackbox logging functionality.

Prerequisites

  1. Scout firmware flashed and running
  2. Serial connection established (115200 baud)
  3. Device shows CLI prompt after boot

Test Procedure

Step 1: Verify Blackbox Initialization

blackbox status

Expected Result:

Blackbox Status: INITIALIZED
Mission Status: INACTIVE
Logging: DISABLED (no active mission)
Storage: FATFS partition mounted

Step 2: Start Mission Logging

blackbox start test_mission_001

Expected Result:

Starting mission: test_mission_001
Mission logging started successfully
Mission ID: test_mission_001
Use 'blackbox status' to monitor logging
Use 'mission evidence <type>' to trigger evidence collection

Step 3: Verify Active Mission

blackbox status

Expected Result:

Blackbox Status: INITIALIZED
Mission Status: ACTIVE
Current Mission: test_mission_001
Logging: ENABLED (events being recorded)

Step 4: Trigger Evidence Collection

mission evidence sensor_test "Manual validation test"

Expected Result:

Collecting evidence for incident: sensor_test
Context: Manual validation test
Evidence collection completed successfully
Evidence logged to current mission log for forensic analysis

Step 5: Wait for Sensor Summaries

Wait 10 seconds to allow automatic sensor summaries to be logged (every 2 seconds).

Step 6: Stop Mission

blackbox stop

Expected Result:

Stopping mission: test_mission_001
Mission logging stopped successfully
Mission logs available via 'mission list'
Use 'mission replay test_mission_001' to view recorded timeline

Step 7: List Available Missions

mission list

Expected Result:

Available Mission Logs:
Mission ID Start Time Duration Size Integrity
-------------------- --------------- ---------- ---------- ----------
test_mission_001 MM-DD HH:MM unknown 1-10KB ok

Step 8: Replay Mission Timeline

mission replay test_mission_001

Expected Result:

Mission Timeline: test_mission_001
Time Type Severity Details
------------ --------------- ---------- --------------------------------------------------
0.0s mission_start info Mission test_mission_001 started
X.Xs evidence_collection critical Event logged
X.Xs sensor_summary info Sensor readings collected
...
Replay complete: X events processed

Step 9: Test Summary Mode

mission replay test_mission_001 --summary

Expected Result:

Mission Timeline: test_mission_001
Mode: Summary (similar events grouped)
Time Type Severity Details
------------ --------------- ---------- --------------------------------------------------
0.0s mission_start info Mission test_mission_001 started
X.Xs evidence_collection critical Event logged
(+ N more sensor_summary events)
...

Step 10: Test Export with HMAC (Optional)

First set an HMAC key:

hmac set test_key 0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef

Then export mission:

mission export test_mission_001

Expected Result:

# Mission Export: test_mission_001
# Format: NDJSON with HMAC-SHA256 integrity footer
# Generated: XXXXXXXXXX
{mission NDJSON data...}
# --- Export Metadata ---
{
"algo": "HMAC-SHA256",
"keyLabel": "test_key",
"signature": "abcdef123456...",
"size_bytes": XXX,
"export_timestamp": XXXXXXXXXX
}
# Export complete with HMAC-SHA256 integrity signature

Test Validation Checklist

  • Blackbox service initializes successfully
  • Mission start/stop commands work correctly
  • Evidence collection triggers and logs data
  • Sensor summaries are automatically logged every 2 seconds
  • Mission list shows created mission logs
  • Mission replay reconstructs timeline correctly
  • Summary mode groups similar events
  • HMAC export generates valid signatures (if key configured)
  • All CLI commands provide appropriate error handling

Expected Log Events

A successful test should generate these event types in the mission log:

  1. mission_start - Mission initiation
  2. evidence_collection - Manual evidence collection
  3. sensor_summary - Periodic sensor readings (every 2 seconds)
  4. mission_stop - Mission completion

Troubleshooting

Issue: Blackbox shows "NOT INITIALIZED" Solution: Check FATFS partition mounting, verify storage configuration

Issue: Mission commands fail Solution: Ensure mission is started before evidence collection

Issue: No sensor data in timeline Solution: Verify sensor drivers are initialized and functioning

Issue: HMAC export shows no signature Solution: Set HMAC key first using hmac set <label> <key_hex>

Performance Expectations

  • Mission start/stop: < 1 second
  • Evidence collection: < 2 seconds
  • Timeline replay: < 5 seconds for typical mission
  • NDJSON export: < 3 seconds with HMAC computation
