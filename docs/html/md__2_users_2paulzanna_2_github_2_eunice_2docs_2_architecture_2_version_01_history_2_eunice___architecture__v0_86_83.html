Eunice v0.6.3 Architecture — Security & Privacy

Date: 12-08-2025
Owner: Paul Zanna
Status: Implementation-Ready

1) Executive Summary

v0.6.3 formalises security, privacy, and licensing: sandboxed connector execution, secrets management, licence scanning, offline mode, and provenance reports. Threat model and policy gates are codified into CI.

2) Scope

Goals

  • Sandboxing presets for connectors/tools (filesystem/network restrictions).
  • Secret handling and rotation; never persisted in artefacts/logs.
  • Licence scanner for PDFs/HTML with policy (allow/deny) gates.
  • Offline mode: cache‑only operation with clear failure states.
  • Run‑level provenance report (sources, licences, hashes).

Non‑Goals (now)

  • Enterprise SSO; remote KMS (documented hooks only).

3) High‑Level Architecture

flowchart LR
A[Policy Config] --> B[Security Manager]
B --> C[Sandbox Launcher]
B --> D[Secrets Vault]
E[Licence Scanner] --> F[Policy Gate]
G[Pipeline Stages] --> F
H[Provenance Reporter] --> I[Report Artifact]

Key properties

  • Security as code: policies in repo; CI enforces.

4) Components

4.1 Security Manager

  • Applies policy to stage launches (e.g., no net/no write outside artefact dir).

4.2 Secrets Vault

  • Local encrypted store (file‑based) with in‑memory injection to processes.

4.3 Licence Scanner & Policy Gate

  • Extracts licence text/flags; blocks use when policy disallows; records decisions.

4.4 Offline Mode

  • Forces connectors to read from cache; surfaces actionable errors when misses occur.

4.5 Provenance Reporter

  • Aggregates source list, hashes, licences, prompts, model IDs into a report.

6) Interfaces

6.1 CLI

  • eunice policy check --run <id>
  • eunice offline enable|disable
  • eunice secrets set --key X --value ***
  • eunice provenance export --run <id>

6.2 Internal FastAPI

POST /policy/check -> {run_id} -> {decisions}
POST /secrets/set -> {key, value}
GET /provenance/{run_id} -> {report}

7) Observability

  • Artefacts: provenance.json, policy_report.json.
  • Metrics: denied actions, offline cache miss rate.

8) Testing & CI

  • Sandbox tests: blocked net/file ops behave as expected.
  • Secrets leak tests: grep logs/artefacts for keys → must be negative.
  • Licence policy tests with synthetic licences.

9) Acceptance (DoD)

  • Policy gate can block runs with disallowed licences.
  • Offline mode functional; provenance report comprehensive.
