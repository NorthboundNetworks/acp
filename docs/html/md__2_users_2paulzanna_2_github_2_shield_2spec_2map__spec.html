Overview

This interface mimics a sci-fi command & control dashboard with a real-time tactical map, system telemetry, live video feeds, and operator status. The UI must be responsive, dark-themed, and visually styled using Tailwind CSS with SVG and/or Canvas overlays.

ðŸ§± Component Spec List

1. <HeaderStatus />

Description: Displays system state, mission time, and network connection info.

Props:

  • status: "ONLINE" | "OFFLINE"
  • missionTime: ISO timestamp or seconds
  • connectionStrength: 0â€“100

State:

  • Auto-increment mission time counter

Behaviour:

  • Animate ONLINE indicator with pulsing light
  • Countdown/clock updates in real-time

2. <UserCommPanel />

Description: Shows operator profile image, name, and comms activity.

Props:

  • operatorName: string
  • avatarUrl: string
  • commsActive: boolean

State:

  • Optional message log preview

Behaviour:

  • Highlight border when commsActive === true
  • Tooltip with last transmission

3. <MetricPanel />

Description: Visual indicators for power, drone count, shield status, and more.

Props:

  • metrics: array of { name, value, icon, maxValue }

Behaviour:

  • Render each metric as an icon + bar or gauge
  • Use colour to indicate danger thresholds (e.g. red below 25%)

Style:

  • Compact, stacked sidebar panel
  • Icons use SVG or font

4. <MapView />

Description: Central tactical display with terrain map, overlaid grid, and danger zone.

Props:

  • mapImageUrl: background image
  • gridSize: number of horizontal and vertical divisions
  • highlightZones: array of polygons or bounding boxes

Behaviour:

  • Draw red danger zones
  • Render live target or unit positions with SVG or <canvas>
  • Respond to zoom/pan (optional)

Implementation:

  • Use absolute positioning and z-index layering
  • Grid rendered with SVG <line> elements
  • Danger zones as <polygon> or styled <div> overlays

5. <ZoneOverlay />

Description: Visual overlay for red restricted zones on map.

Props:

  • zones: array of [[x1, y1], [x2, y2], ...] (polygon points)

Behaviour:

  • Draw filled red shapes with transparency over map
  • Animate appearance/fade if entering zone

6. <GridOverlay />

Description: Static grid to divide map into sectors.

Props:

  • rows: number
  • cols: number

Behaviour:

  • Render vertical and horizontal lines in evenly spaced intervals
  • Optional sector labels (A1, A2...)

Implementation:

  • Use SVG with grid snapping
  • Should not interfere with other interactive elements

7. <FeedPanel />

Description: Right-side panel for displaying live video or static feed.

Props:

  • feeds: array of { id, src, label, isActive }

Behaviour:

  • Active feed is highlighted
  • Can be a <video> or <img> tag
  • Add toggle buttons or scrollable list of feeds

8. <ToggleSwitch />

Description: Custom sci-fi toggle switch component.

Props:

  • label: string
  • value: boolean
  • onToggle: callback

Style:

  • Industrial/sci-fi toggle with LED-style indicators
  • Debounced animation on toggle

9. <AlertBanner />

Description: Bottom bar for alert messages or mission updates.

Props:

  • alerts: array of { type, message, timestamp }

Behaviour:

  • Scroll or cycle alerts if multiple active
  • Color-coded by type: red (critical), yellow (warning), blue (info)

ðŸŽ¨ Style Guidelines

  • Use a dark theme with neon accents (blue, red, green)
  • Use Tailwind CSS for layout and spacing
  • All icons should be SVG, ideally from a central icon set
  • Font: custom sci-fi font or a fallback monospace (Orbitron, Share Tech Mono, etc.)
