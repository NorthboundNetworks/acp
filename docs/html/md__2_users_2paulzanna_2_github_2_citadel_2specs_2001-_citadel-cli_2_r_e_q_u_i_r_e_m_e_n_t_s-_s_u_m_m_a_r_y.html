Requirements Summary - Citadel CLI v0.1

Purpose: Address code quality checklist gaps by documenting detailed build, style, and quality requirements.

C11 Compliance Requirements

Standard: ISO/IEC 9899:2011 (C11) with POSIX.1-2008 extensions

Compiler Support:

  • GCC ≥ 7.0 (Linux)
  • Clang ≥ 6.0 (Linux/macOS)
  • Xcode Command Line Tools ≥ 10.0 (macOS)

Language Features Used:

  • stdint.h fixed-width types (uint8_t, uint16_t, etc.)
  • stdbool.h boolean type
  • Designated initializers for structs
  • Static assertions (_Static_assert)
  • Anonymous structs/unions (if needed)

Forbidden Features (for portability):

  • VLA (Variable Length Arrays) - use fixed-size or malloc
  • __attribute__ extensions except packed and aligned
  • GCC/Clang-specific builtins

Compiler Flags Specification

Release Build

CFLAGS = -std=c11 -O2 -g -Wall -Wextra -Werror -Wpedantic \
-Wformat=2 -Wformat-security -Wconversion -Wsign-conversion \
-Wcast-align -Wstrict-prototypes -Wmissing-prototypes \
-Wredundant-decls -Wnested-externs -Winline \
-fstack-protector-strong -D_FORTIFY_SOURCE=2 -fPIE
LDFLAGS = -Wl,-z,relro,-z,now -pie # Linux
LDFLAGS = -Wl,-bind_at_load # macOS

Debug Build

DEBUG_CFLAGS = -std=c11 -O0 -g3 -Wall -Wextra -Werror \
-fsanitize=address -fsanitize=undefined \
-fno-omit-frame-pointer -DDEBUG=1
DEBUG_LDFLAGS = -fsanitize=address -fsanitize=undefined

Memory Safety Requirements

Allocation Patterns:

  • Prefer stack allocation for small, bounded data
  • Use malloc()/free() pairs with explicit size tracking
  • No alloca() or VLA for security reasons
  • Initialize all allocated memory to zero

Buffer Management:

  • All string operations use strn*() family functions
  • Buffer sizes validated before use
  • Circular buffer indices checked for overflow
  • CRC calculations use bounds-checked loops

Validation Requirements:

  • Valgrind clean execution (Linux)
  • AddressSanitizer clean execution (all platforms)
  • No memory leaks in normal operation
  • Graceful handling of allocation failures

Code Style Standards

Naming Conventions

  • Functions: snake_case (e.g., serial_open, scp_decode_frame)
  • Variables: snake_case (e.g., frame_count, rx_buffer)
  • Types: snake_case_t suffix (e.g., telemetry_frame_t)
  • Constants: UPPER_CASE (e.g., MAX_FRAME_SIZE, DEFAULT_BAUD)
  • Macros: UPPER_CASE with descriptive names

File Organization

// Header include order:
#include <system_headers.h> // Standard C library
#include <posix_headers.h> // POSIX extensions
#include <third_party.h> // External libraries
#include "local_headers.h" // Project headers
// Function order in .c files:
// 1. Static/local functions
// 2. Public interface functions
// 3. Main/entry functions

Error Handling Patterns

// Return codes: 0 = success, negative = error
int serial_open(const char* port, int baud, int* fd);
// Error checking pattern:
int result = serial_open("/dev/ttyUSB0", 115200, &fd);
if (result < 0) {
fprintf(stderr, "Failed to open serial port: %s\n", strerror(-result));
return EXIT_FAILURE;
}

Testing Requirements

Unit Test Coverage

  • Core functions: SCP decoder, frame parser, config loader
  • Utility functions: ISO 8601 formatting, CRC calculation
  • Edge cases: Buffer overflow, malformed frames, config errors
  • Target: ≥80% line coverage for src/ directory

Integration Tests

  • Serial port simulation using pty pairs
  • End-to-end frame processing with mock data
  • CLI argument parsing and precedence rules
  • Graceful shutdown and error recovery

Memory Testing

# Valgrind (Linux)
valgrind --leak-check=full --show-leak-kinds=all ./citadel_cli_test
# AddressSanitizer (all platforms)
./citadel_cli_test # Built with -fsanitize=address

Dependencies Management

Required System Libraries

  • libc: C11 standard library with POSIX extensions
  • termios: POSIX serial port control (part of libc)

Optional Dependencies

  • ncurses: TUI mode (graceful fallback to ASCII if missing)
  • pkg-config: Build-time dependency detection

Vendored Dependencies

  • Unity: Single-file C test framework (tests/unity/)
  • No runtime dependencies on vendored code

Build System Requirements

Makefile Compatibility

  • POSIX make compatibility (no GNU make extensions)
  • Support for parallel builds (make -j)
  • Cross-compilation support via CC/CFLAGS environment
  • Automatic dependency generation for headers

Installation

make install PREFIX=/usr/local # Default prefix
make install PREFIX=/opt/citadel DESTDIR=/tmp/staging

Distribution

  • Static binary preferred for deployment
  • Checksums (SHA-256) for all release artifacts
  • Build reproducibility via SOURCE_DATE_EPOCH

Security Hardening

Compile-Time Mitigations

  • Stack protection: -fstack-protector-strong
  • Buffer overflow detection: -D_FORTIFY_SOURCE=2
  • Position-independent executable: -fPIE -pie
  • Format string protection: -Wformat-security

Runtime Mitigations

  • Address space layout randomization (ASLR) support
  • Stack canaries for function protection
  • Immediate binding: -Wl,-z,now
  • Read-only relocations: -Wl,-z,relro

Input Validation

  • Serial data bounds checking before buffer writes
  • Frame length validation before parsing
  • CRC verification for frame integrity
  • Config file parsing with size limits

Quality Gates

Pre-commit Checks

  1. Clean compilation with all warnings enabled
  2. Static analysis passes (clang-tidy + cppcheck)
  3. Unit tests pass with memory checking
  4. Code formatting consistency check

Release Criteria

  1. All functional requirements implemented and tested
  2. Performance benchmarks meet specification (p95 ≤ 200ms)
  3. Memory usage within bounds (≤ 50MB RSS)
  4. Security review passed for critical paths
  5. Documentation complete and accurate
