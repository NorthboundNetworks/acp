Simplified SCP Frame Structure (MVP)

Based on SCP v1.1 tlm_status_t message type, simplified for CLI MVP:

Field Size Description CLI Mapping
SYNC 2 bytes 0xAA 0x55 Frame detection
MsgType 1 byte 0x10 (tlm_status) Message validation
Length 2 bytes Payload size Frame boundary
armed 1 byte Arm status Not displayed (MVP)
mode 1 byte Flight mode Not displayed (MVP)
gps_fix 1 byte GPS fix type → gps_fix (0=NONE, 2=2D, 3=3D)
battery_percent 1 byte Battery % → battery_v (scaled to voltage)
roll/pitch/yaw 6 bytes Attitude (deg×100) → imu_gyro_z (yaw only)
uptime_ms 4 bytes System uptime → cpu_load_pct (derived)
timestamp_ms 4 bytes Timestamp → timestamp_utc (converted)
CRC16 2 bytes Frame checksum Validation only

Mock Field Mappings

  • cpu_load_pct = (uptime_ms % 10000) / 100.0 (cycling 0-100%)
  • cpu_temp_c = 45.0 (fixed mock value)
  • battery_v = battery_percent * 0.12 + 10.0 (scale to 10.0-22.2V range)
  • imu_gyro_z = yaw * 0.01745329 (deg/100 → rad/s)
  • gps_fix = gps_fix == 0 ? "NONE" : (gps_fix == 2 ? "2D" : "3D")
  • timestamp_utc = Convert timestamp_ms to ISO 8601 UTC format

Note: This simplified approach skips SCP v1.1 authentication and complex message routing for MVP.
