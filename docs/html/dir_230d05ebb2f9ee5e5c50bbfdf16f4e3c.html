Detailed Description

Citadel Core CLI

![Build Status](https://img.shields.io/badge/build-passing-brightgreen) ![C Standard](https://img.shields.io/badge/C-C11-blue.svg) ![Platform](https://img.shields.io/badge/platform-Linux%20%7C%20macOS%20%7C%20Windows-lightgrey) ![License](https://img.shields.io/badge/license-MIT-blue.svg)

A high-performance, cross-platform command-line interface for real-time telemetry monitoring and Scout drone communication using the SCP (Serial Communication Protocol) v1.1.

Features

  • 🚀 Real-time Telemetry: Live monitoring of drone CPU, battery, GPS, and sensor data
  • 📡 SCP Protocol: Full implementation of SCP v1.1 for reliable serial communication
  • 🖥️ Dual Display Modes: ASCII text output or full-color ncurses TUI
  • 📊 CSV Logging: Structured data logging for analysis and replay
  • ⚡ High Performance: Sub-microsecond latency with 9M+ ops/sec throughput
  • 🔒 Enterprise Ready: Static analysis, SBOM generation, and reproducible builds
  • 🌐 Cross Platform: Native support for Linux, macOS, and Windows

Quick Start

# Clone and build
git clone https://github.com/your-org/citadel.git
cd citadel
make
# Connect to drone and start monitoring
./bin/citadel_cli --port /dev/ttyUSB0 --baud 115200 --mode tui
# View help
./bin/citadel_cli --help

Installation

Prerequisites

  • C11 Compiler: GCC 7+ or Clang 9+
  • Make: GNU Make or compatible
  • ncurses (optional): For TUI mode
  • POSIX System: Linux, macOS, or Windows with WSL/MinGW

Building from Source

# Clone repository
git clone https://github.com/your-org/citadel.git
cd citadel
# Build release version
make release
# Build with debug symbols
make DEBUG=1
# Run tests
make test
# Generate performance benchmarks
make perf
# Create release package
make release-package

Installation Options

# System-wide installation (requires sudo)
sudo make install
# Custom prefix
make install PREFIX=/opt/citadel
# Portable installation
cp bin/citadel_cli /usr/local/bin/

Usage

Basic Operation

# Connect to drone via serial port
citadel_cli --port /dev/ttyUSB0 --baud 115200
# Enable TUI mode with CSV logging
citadel_cli --port COM3 --baud 115200 --mode tui --log telemetry.csv
# Replay logged data
citadel_cli --replay telemetry.csv --mode tui

Command Line Options

Option Description Default
--port <device> Serial port device /dev/ttyUSB0
--baud <rate> Baud rate (9600-921600) 115200
--mode <ascii\|tui> Display mode ascii
--log <filename> CSV log file disabled
--config <file> Configuration file citadel.conf
--timeout <ms> Connection timeout 5000
--replay <file> Replay mode from CSV disabled
--help Show help message
--version Show version info

Configuration File

Create citadel.conf for persistent settings:

[serial]
port = /dev/ttyUSB0
baud = 115200
timeout = 5000
[display]
mode = tui
update_rate = 30
colors = true
[logging]
enabled = true
filename = logs/citadel_%Y%m%d_%H%M%S.csv
max_size = 100MB
rotate = true
[scp]
validate_crc = true
sync_timeout = 1000
max_retries = 3

Architecture

Core Components

  • Serial Interface (serial.c): Cross-platform serial communication
  • SCP Protocol (scp.c): Frame encoding/decoding with CRC validation
  • Display Engine (display_*.c): ASCII and ncurses rendering
  • CSV Logger (log.c): Structured data logging with rotation
  • Configuration (config.c): INI-style configuration management

Data Flow

┌─────────────┐ Serial ┌──────────────┐ SCP ┌─────────────┐

│
▼
┌─────────────┐ Telemetry ┌──────────────┐ Display ┌─────────────┐
│ CSV Logger │←──────────────│ Data Buffer │───────────→│ Renderer │
