The SHIELD database is a SQLite database designed to support geopolitical analysis, news monitoring, stability metrics, and alert systems. The database contains 7 tables with a total of 1,188 records and maintains relationships between countries and various analytical data points.

Key Features

  • Financial Indicators: Economic data tracking (ready for implementation)
  • Extensible Design: Modular structure for additional data types

1. üåç Countries Table

Purpose: Master country data with demographics and peace index
Records: 250 countries

CREATE TABLE countries (
id INTEGER NOT NULL PRIMARY KEY,
code VARCHAR(2) UNIQUE, -- ISO 2-letter country code
name VARCHAR, -- Official country name
region VARCHAR, -- Geographic region (e.g., "Europe")
subregion VARCHAR, -- Geographic subregion (e.g., "Western Europe")
population INTEGER, -- Population count
area FLOAT, -- Area in square kilometers
gdp FLOAT, -- Gross Domestic Product
peace_index FLOAT, -- Global Peace Index score
created_at DATETIME -- Record creation timestamp
);
-- Indexes
CREATE INDEX ix_countries_name ON countries (name);
CREATE INDEX ix_countries_id ON countries (id);
CREATE UNIQUE INDEX ix_countries_code ON countries (code);

Sample Data Structure:

  • Code: "US", "FR", "JP", etc.
  • Name: "United States", "France", "Japan", etc.
  • Region: "Americas", "Europe", "Asia", etc.
  • Population: Actual population numbers
  • GDP: Economic data in USD
  • Peace Index: Lower values = more peaceful

3. üì∞ News Articles Table

Purpose: News articles for geopolitical analysis with sentiment scoring
Records: 6 articles (sample data)

CREATE TABLE news_articles (
id VARCHAR NOT NULL PRIMARY KEY,
title VARCHAR, -- Article headline
description TEXT, -- Article summary/excerpt
content TEXT, -- Full article content
url VARCHAR UNIQUE, -- Source URL (unique constraint)
source VARCHAR, -- News source (e.g., "Reuters", "BBC")
country_code VARCHAR(2), -- Related country (FK to countries.code)
category VARCHAR, -- Article category (e.g., "politics", "security")

);
-- Indexes
CREATE INDEX ix_news_articles_country_code ON news_articles (country_code);
CREATE INDEX ix_news_articles_title ON news_articles (title);

Features:

  • Sentiment Analysis: Automated scoring of article tone
  • Relevance Scoring: Importance for geopolitical analysis
  • Country Linking: Direct association with country codes
  • Duplicate Prevention: Unique URL constraint

5. üìà Stability Metrics Table

Purpose: Calculated stability scores for countries

CREATE TABLE stability_metrics (
