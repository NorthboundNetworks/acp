Detailed Description

Mission blackbox logging service implementation.

This service provides comprehensive mission logging capabilities with:

  • Per-flight NDJSON structured logging to FATFS flash partition
  • Background writer task for non-blocking real-time operation
  • Keep-last-1 rotation policy for storage management
  • HMAC-SHA256 signatures for tamper-evident log exports
  • Evidence collection with contextual metadata
  • Mission timeline replay functionality

Architecture:

  • Event producer API for real-time logging without blocking
  • Ring buffer queue between producers and background writer
  • FATFS filesystem for mission log persistence
  • Structured NDJSON format for machine parsing and human readability

Performance characteristics:

  • Event queuing: <100us typical latency
  • Background writes: 2-5ms per batch (non-blocking to control loop)
  • Storage efficiency: ~2-5KB per minute of flight time
  • Queue capacity: 256 events before backpressure

Constitution compliance:

  • No blocking operations in producer API (real-time requirement)
  • Background task priority lower than control loops
  • Memory bounds enforced (no dynamic allocation in hot paths)
  • Performance impact <10% CPU overhead per validation tests
