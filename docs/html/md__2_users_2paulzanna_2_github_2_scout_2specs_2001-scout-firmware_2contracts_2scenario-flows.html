Arming Flows (User Story 1)

  • Primary flow: CONFIG mode → attempt arm → rejected with reason; switch to FLIGHT → preflight passes → arm succeeds.
  • Alternate flow: FLIGHT → preflight fails (e.g., CPU >70%) → arming rejected with reason.
  • Exception flow: Armed → receiver loss → failsafe disarm ≤1 s from loss detection.

Loss-of-Link (User Story 2)

  • Pre-arm: loss of link keeps system disarmed; preflight indicates receiver fault.
  • Post-arm: loss detection after 1.0 s without valid control/heartbeat → disarm ≤1.0 s; STATUSTEXT on recovery.

Configuration Import Errors (FR-004)

  • Corrupted dump: reject import with error; existing configuration remains unchanged; report first invalid key and reason.
  • Incomplete dump: unknown keys ignored; missing keys retain current values; final config remains coherent.

Rapid-Motion Vehicle Launch (FR-013)

  • Within envelope: speed ≤ 80 km/h and |roll|,|pitch| ≤ 15° with other gates OK → allow arming/mode entry; monitor for sustained violations.
  • Outside envelope: arming/mode entry rejected with STATUSTEXT; remain in safe mode.
  • False positives: use windowing averages (1 s speed, 500 ms attitude) to avoid transient misclassification.

No GPS Fix Behaviors (FR-017)

  • Pre-arm: arming permitted if GPS not required; GPS-dependent features disabled.
  • Post-arm: if NO FIX arises, disable GPS behaviors; maintain non-GPS modes; indicate NO FIX in status and STATUSTEXT.
