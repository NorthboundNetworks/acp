Eunice v0.5.10 Architecture — Evaluation Harness

Date: 12-08-2025
Owner: Paul Zanna
Status: Implementation-Ready

1) Executive Summary

v0.5.10 adds a repeatable evaluation harness to quantify output quality and runtime characteristics. It measures faithfulness/attribution, coverage, and latency/cost, with a small Human Rater UI for spot checks. CI gates enforce thresholds to prevent regressions.

2) Scope

Goals

  • Fixed benchmark topics with gold artefacts (citations, anchors, expected coverage lists).
  • Automated metrics: Attribution Precision/F1, **Coverage@K**, Claim‑level Faithfulness (quote match to anchor), Latency/Cost per stage.
  • Human rater UI (local web or TUI) to label sampled claims.
  • CI pipeline to fail below thresholds; publish evaluation artefacts.

Non‑Goals (now)

  • Full crowdsourcing platform; multi‑tenant web app.

3) High‑Level Architecture

Key properties

  • Deterministic: same inputs → same metrics.
  • Pluggable metrics; JSON outputs snapshot‑tested.

4) Components

4.1 Benchmarks Registry

  • YAML/JSON index of benchmark topics with: input prompts, expected bibliographies, seed, model id.

4.2 Evaluator

  • Loads pipeline outputs; computes metrics; writes eval_report.json and per‑metric CSVs.

4.3 Human Rater UI

  • Minimal local web/TUI to display sampled assertions with source snippets and capture faithfulness labels.

4.4 Metrics Store & Reporter

  • Persists eval_run rows and per‑metric details; CI reporter summarises thresholds vs results.

5) Metrics (definitions)

  • Attribution Precision (AP): fraction of cited anchors that actually contain/support the asserted text (sampling via quote‑match + human check).
  • Attribution F1: harmonic mean of AP and recall (fraction of sampled assertions with at least one correct anchor).
  • **Coverage@K**: overlap between generated citations and a curated gold bibliography (Jaccard).
  • Latency/Cost: per‑stage p50/p95 latency; token/compute cost; cache hit ratio.

7) Interfaces

7.1 CLI

  • eunice eval run --benchmark <id>
  • eunice eval report --run <eval_run_id>
  • eunice eval ui

7.2 Internal FastAPI

POST /eval/run -> {benchmark_id} -> {eval_run_id}
GET /eval/{id} -> {metrics, report}
POST /eval/label -> {eval_run_id, assertion_id, label} -> {ok}

8) Observability

  • Artefacts: eval_report.json, metrics.csv, labels.csv.
  • CI badge: pass/fail with threshold summary.

9) Testing & CI

  • Golden benchmark snapshots (metrics stable).
  • Synthetic corruption tests (e.g., broken anchors) must reduce AP as expected.
  • Determinism test: same inputs → identical eval_report.json.

10) Acceptance (DoD)

  • CI fails when thresholds not met (configurable).
  • Rater UI writes labels and updates metrics.
  • Metric exports reproducible and stored with run manifests.
