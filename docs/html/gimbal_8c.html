Detailed Description

Two-axis gimbal control service implementation.

This service provides comprehensive gimbal attitude control with three operating modes optimized for different mission requirements. Supports CLI operator control and SCP-based external command integration.

Operating Modes:

  • FIXED: Hold specific pitch/yaw angles with servo feedback
  • FOLLOW_HEADING: Track GPS heading while maintaining pitch
  • SEARCH_SWEEP: Automated scanning pattern for surveillance

Hardware Interface:

  • PWM servo control via ESP32-S3 LEDC peripheral (50Hz standard)
  • Two-axis control: pitch (±90°) and yaw (±180°)
  • Hardware-accelerated PWM generation for smooth motion
  • GPIO pins 14 (pitch) and 12 (yaw) - configurable

Control Characteristics:

  • Update rate: 50Hz for smooth motion (20ms intervals)
  • Servo timing: 1-2ms pulse width, 20ms period (standard RC servo)
  • Angle resolution: ~0.1° with 14-bit PWM resolution
  • Slew rate limiting: 45°/s default (configurable for smooth movement)

Performance:

  • Command latency: <300ms from CLI/SCP to servo movement
  • Angle accuracy: ±2° in fixed mode, ±3° in follow mode
  • Sweep precision: Configurable rate and bounds (±135° default)
  • CPU overhead: ~4% including servo updates and mode logic

Integration:

  • CLI commands: mode, set, sweep, status for operator control
  • External control via SCP gateway
  • GPS integration: Follow heading mode uses GPS course data
  • Blackbox logging: All gimbal events logged for mission replay

Constitution Compliance:

  • Real-time: Non-blocking operations, background servo updates
  • Safety: Angle bounds enforced, slew rate limiting prevents damage
  • Performance: <300ms latency requirement met, <5% CPU overhead
  • Testability: Comprehensive status reporting and mode validation
