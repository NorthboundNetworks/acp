CLI Golden Transcripts Test Suite

Purpose

Validate CLI command behavior with golden transcripts covering round-trip operations, error cases, and parameter validation per CLI contracts.

Test Environment

  • Hardware: ESP32-S3 development board
  • Connection: USB serial at 115200 baud
  • Mode: CONFIG mode for safe CLI testing
  • Prerequisites: Propellers removed, bench setup only

Test Cases

TC001: Version Command

Test: Basic version query Expected: Firmware version, API version, build info

> version
firmware: 1.0.0-dev
api: 1.0.0
build: abc123ef/2025-10-26

TC002: Parameter Get/Set Round-trip

Test: Get default, set new value, verify change, round-trip Expected: Parameter validation and persistence

> get flight.max_altitude_m
flight.max_altitude_m=30
> set flight.max_altitude_m 50
OK
> get flight.max_altitude_m
flight.max_altitude_m=50
> save
SAVED
> reboot
REBOOTING
[wait for boot]
> get flight.max_altitude_m
flight.max_altitude_m=50

TC003: Invalid Parameter Handling

Test: Unknown parameter and invalid values Expected: Clear error messages

> get nonexistent.param
Error: Unknown parameter 'nonexistent.param'
> set flight.max_altitude_m 999
Error: Invalid value '999' for parameter 'flight.max_altitude_m'
> set system.boot_count 42
Error: Parameter 'system.boot_count' is read-only
