Eunice v0.6.5 Architecture — Backend Task Workers & Processing Logic

Date: 14-08-2025
Owner: Paul Zanna
Status: Planning

1) Executive Summary

v0.6.5 implements intelligent backend task workers that execute research workflows autonomously. This includes literature processing, experiment design generation, materials automation, and result synthesis with progress tracking and error recovery.

2) Scope

Goals

  • Task Worker Engine: Autonomous processing of literature_review, analysis, synthesis tasks
  • Literature Automation: Multi-source search, filtering, and content extraction
  • Experiment Design Generation: Automated DoE creation with statistical validation
  • Materials Processing: BOM generation, cost analysis, and supplier integration
  • Progress Tracking: Real-time status updates with detailed progress indicators
  • Error Recovery: Robust retry mechanisms with exponential backoff
  • Result Storage: Structured output with metadata and provenance tracking

Non-Goals (this version)

  • Multi-tenant worker isolation
  • Distributed processing across multiple machines
  • Custom user-defined task types

6) Worker Implementation Specifications

6.1 Literature Worker Workflow

class LiteratureWorker:
async def process_literature_task(self, task_data):
# 1. Parse search parameters
sources = task_data.payload.literature.sources
filters = task_data.payload.literature.filters
# 2. Execute multi-source search
await self.update_progress(10, "Initiating searches")
results = await self.search_all_sources(sources, filters)
# 3. Content extraction and processing
await self.update_progress(40, "Extracting content")
processed = await self.extract_content(results)
# 4. Relevance scoring and filtering
await self.update_progress(70, "Scoring relevance")
filtered = await self.score_and_filter(processed)
# 5. Generate summary and insights
await self.update_progress(90, "Generating insights")
summary = await self.generate_summary(filtered)
# 6. Store results
await self.store_results(summary, filtered)
await self.update_progress(100, "Complete")

6.2 Experiment Design Worker Workflow

class ExperimentWorker:
async def process_experiment_task(self, task_data):
# 1. Parse design parameters
template = task_data.payload.experiment.template
factors = task_data.payload.experiment.factors
responses = task_data.payload.experiment.responses
# 2. Generate experimental design
await self.update_progress(20, "Generating design matrix")
design = await self.generate_design(template, factors)
# 3. Statistical validation
await self.update_progress(50, "Validating design")
validated = await self.validate_design(design, responses)
# 4. Protocol generation
await self.update_progress(80, "Generating protocols")
protocol = await self.generate_protocol(validated)
# 5. Store results
await self.store_design_results(validated, protocol)
await self.update_progress(100, "Complete")

Integration Requirements

  • Error messages are actionable for users
  • Performance metrics are available in monitoring dashboard

12) Risk Assessment

High Risk

  • External API Rate Limits: Implement aggressive caching and request queuing
  • Worker Memory Leaks: Implement worker recycling and monitoring
  • Database Deadlocks: Use proper transaction isolation and retry logic

Medium Risk

  • Network Failures: Implement circuit breakers and timeout handling
  • Large Result Sets: Use streaming and pagination for memory efficiency
  • Queue Overflow: Implement priority queues and load shedding

Low Risk

  • Configuration Drift: Use infrastructure as code and configuration validation
  • Log Storage: Implement log rotation and archival policies
