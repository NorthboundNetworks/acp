Eunice v0.6.1 Architecture — Protocol Generator & Costing

Date: 12-08-2025
Owner: Paul Zanna
Status: Implemented

1) Executive Summary

v0.6.1 converts synthesis outputs into versioned laboratory protocols (“lab playbooks”) with materials/BOM, stepwise procedures, timings, and hazards. It adds costing with supplier alternatives and local‑sourcing notes, keeping full provenance to evidence anchors.

2) Scope

Goals

  • Protocol generator templating: SOP headers, purpose, materials, equipment, steps, QC checks, hazards.
  • BOM with cost lookups and alternatives; attach suppliers (local configurable catalogues).
  • Versioned playbooks with semantic diffs and provenance back to claims/anchors.
  • Export formats: Markdown/PDF/CSV (BOM) with stable IDs.

Non‑Goals (now)

  • Live purchasing; inventory management.

3) High‑Level Architecture

flowchart LR
A[Synthesis Blocks] --> B[Protocol Planner]
B --> C[Protocol Renderer]
B --> D[BOM Builder]
D --> E[Costing Engine]
C --> F[Playbook Store]
E --> F
G[Supplier Catalogues (local JSON/CSV)] --> E

Key properties

  • Deterministic protocol generation from synthesis + templates.
  • Costs pluggable by catalogue; offline‑friendly.
    • Optional USD⇄AUD costing conversions via env: EUNICE_FX_USD_AUD / EUNICE_FX_AUD_USD.

4) Components

4.1 Protocol Planner

  • Maps evidence‑backed steps; expands with timings, temperatures, and QC checkpoints based on templates.

4.2 BOM Builder

  • Normalises materials/equipment; deduplicates items; assigns units; generates alternatives.

4.3 Costing Engine

  • Resolves items to supplier SKUs (local JSON/CSV); computes totals; flags unavailable items; suggests substitutes.

4.4 Playbook Store

  • Versioned records with diffs; provenance (claim/anchor IDs) retained per step/material.

6) Interfaces

6.1 CLI

  • eunice protocol generate --synthesis <id> --template <file>
  • eunice protocol export --playbook <id> --format md|pdf
  • eunice bom price --playbook <id> --catalog <file>

6.2 Internal FastAPI

POST /protocol/generate -> {synthesis_id, template} -> {playbook_id}
GET /playbook/{id} -> {steps, bom, cost}
POST /bom/price -> {playbook_id, catalog_path} -> {priced_bom}

7) Templates & Catalogues

  • Templates under /templates/protocols/*.json (schema‑validated).
  • Supplier catalogues under /catalogues/*.csv|json with a common schema.

8) Observability

  • Artefacts: playbook.md, bom.csv, cost_summary.json, provenance.json.

9) Testing & CI

  • Snapshot tests for playbook text and BOM.
  • Costing unit tests with sample catalogues.
  • Provenance check: each step/material references claim/anchor when available.

10) Acceptance (DoD)

  • Deterministic playbook generation from same synthesis & template.
  • BOM with costs and alternatives; totals computed.
  • Export formats generated; provenance intact.
