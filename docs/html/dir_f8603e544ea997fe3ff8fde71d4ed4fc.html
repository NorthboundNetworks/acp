Detailed Description

Configuration Management System

Overview

This document describes the centralized configuration management system for the Shield project.

Structure

config/
├── .env # Environment variables (production)
├── .env.example # Template for environment variables
├── config.py # Centralized configuration module
└── countries.json # Country data for geopolitical analysis

Configuration Module (config/config.py)

The configuration module provides a centralized way to manage all application settings and environment variables.

Features

  • Automatic environment variable loading from centralized .env file
  • Fallback to .env.example if main .env file is missing
  • Configuration validation with helpful error messages
  • Function-based configuration getters for better maintainability
  • Environment detection (production vs development)
  • Comprehensive logging

Available Configuration Functions

API Keys and URLs

  • get_news_api_key() - News API key for news collection
  • get_alpha_vantage_api_key() - Alpha Vantage API key for financial data
  • get_world_bank_api_key() - World Bank API key
  • get_rest_countries_api() - REST Countries API URL
  • get_world_bank_api_url() - World Bank API base URL

Scheduling

  • get_news_collection_schedule() - Cron schedule for news collection
  • get_stability_update_schedule() - Cron schedule for stability updates

Database

  • get_database_url() - Database connection URL

Utilities

  • get_config_dir() - Path to configuration directory
  • get_countries_json_path() - Path to countries.json file
  • is_production() - Check if running in production
  • is_development() - Check if running in development
  • validate_config() - Validate critical configuration values

Usage

In Backend Code

# Import centralized configuration
import sys
from pathlib import Path
# Add config directory to path
config_dir = Path(__file__).parent.parent.parent / 'config'
sys.path.insert(0, str(config_dir))
import config as app_config
# Use configuration functions
news_api_key = app_config.get_news_api_key()
countries_api = app_config.get_rest_countries_api()
database_url = app_config.get_database_url()

Environment Variables

The system loads environment variables from config/.env. Key variables include:

# API Keys
NEWS_API_KEY=your_news_api_key_here
ALPHA_VANTAGE_API_KEY=your_alpha_vantage_key_here
WORLD_BANK_API_KEY=your_world_bank_key_here
# API URLs
REST_COUNTRIES_API=https://restcountries.com/v3.1
WORLD_BANK_API_URL=https://api.worldbank.org/v2
# Database
DATABASE_URL=sqlite:///./sql_app.db
# Scheduling (Cron format)
NEWS_COLLECTION_SCHEDULE=*/30 * * * *
STABILITY_UPDATE_SCHEDULE=0 */6 * * *
# Environment
ENVIRONMENT=development

Migration from Previous System

Before (Scattered Configuration)

import os
from dotenv import load_dotenv
load_dotenv('backend/.env')
news_api_key = os.getenv('NEWS_API_KEY', 'default')

After (Centralized Configuration)

import config as app_config
news_api_key = app_config.get_news_api_key()

Benefits

  1. Centralization: All configuration files in one location
  2. Maintainability: Function-based access with clear interfaces
  3. Validation: Automatic validation of critical configuration values
  4. Logging: Comprehensive logging for debugging configuration issues
  5. Fallbacks: Graceful handling of missing configuration files
  6. Type Safety: Clear function signatures for configuration access
  7. Documentation: Self-documenting function names and docstrings
