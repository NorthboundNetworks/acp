Summary

Implement a secure, auditable binary communication protocol (SCP v1.1) as the sole operational control protocol for the Scout ISR drone. The protocol enables role-based command authority, cryptographic authentication, and priority-based message handling for mission-critical communication between ESP32-S3 flight controller and ground control systems in contested military environments.

Technical Context

Language/Version: C (GNU C11) with ESP-IDF 5.x via PlatformIO
Primary Dependencies: ESP-IDF drivers (UART/SPI/I2C), FreeRTOS, existing CLI framework, NEEDS CLARIFICATION: COBS encoding library, HMAC-SHA256 crypto implementation
Storage: NVS (Non-Volatile Storage) for session keys and configuration, existing FATFS partition for audit logs
Testing: Unity test framework for unit tests, NEEDS CLARIFICATION: Hardware-in-the-loop test approach for protocol validation
Target Platform: ESP32-S3 (Freenove ESP32-S3 WROOM) embedded firmware Project Type: Single embedded firmware project with integrated protocol stack
Performance Goals: <100ms command acknowledgment, 1-10Hz telemetry streaming, <70% CPU utilization during operation
Constraints: <8KB memory for protocol buffers, real-time constraints for control loop integration, NEEDS CLARIFICATION: Flash storage limits for cryptographic libraries
Scale/Scope: Support up to 3 concurrent operator sessions, 14 message types, role-based access control for 3 operator types
