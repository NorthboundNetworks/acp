Telemetry Contracts: Baseline Heartbeat/Status & Control

Created: 2025-10-25 Branch: 001-scout-firmware Spec: ../spec.md

Baseline Profile

  • Heartbeat/status at ≥1 Hz
  • Arm/disarm control (subject to safety gates)
  • Fields (suggested): system_status, mode, safety_state (arming_allowed, cpu_budget_ok), pose_summary, battery, link_quality

Compatibility Notes

  • Use a standards‑compliant baseline for heartbeat/status and arm/disarm.
  • Exact message mapping to be finalized during implementation; maintain compatibility notes here.

MAVLink v2 common mapping (Baseline)

  • Protocol: MAVLink v2 common
  • Telemetry rate: ≥1 Hz with jitter ≤±250 ms and ≥99% delivery over 10 minutes

Messages

1) HEARTBEAT (ID: 0)

  • Rate: 1 Hz minimum (target 1–2 Hz)
  • Fields: type, autopilot, base_mode (arming flag), custom_mode (0=hover, 1=loiter, 2=forward), system_status

2) SYS_STATUS (ID: 1)

  • Rate: 1 Hz (aligned with HEARTBEAT)
  • Fields: onboard_control_sensors_present/enabled/health (IMU/MAG/BARO/GPS), load (×10), battery fields if available

3) STATUSTEXT (ID: 253)

  • Event-driven errors/warnings (e.g., preflight gate rejection reasons)

4) COMMAND_LONG (ID: 76) with MAV_CMD_COMPONENT_ARM_DISARM (400)

  • Param1: 1=arm, 0=disarm (subject to gates); respond with COMMAND_ACK and STATUSTEXT on failure

5) SET_MODE (ID: 11)

  • Mode changes via base_mode/custom_mode mapping; reject when constraints fail

Timing & Measurement

  • Timestamps via esp_timer (µs); scheduling via FreeRTOS
  • Loss-of-link declared after 1.0 s without valid control/heartbeat; disarm ≤1.0 s after detection

Acceptance Evidence

  • 10‑minute bench session logs with periodic HEARTBEAT/SYS_STATUS and successful arm/disarm round‑trips.
  • Loss‑of‑link test confirming disarm ≤1 s from loss detection and safe state upon reconnection.
  • Golden logs for mode changes and rejection cases.
