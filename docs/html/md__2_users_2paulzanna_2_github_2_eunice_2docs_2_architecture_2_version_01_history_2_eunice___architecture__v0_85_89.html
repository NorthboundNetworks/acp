Eunice v0.5.9 Architecture — Synthesis Levels (UG/MSc/PhD)

Date: 12-08-2025
Owner: Paul Zanna
Status: Implementation-Ready

1) Executive Summary

v0.5.9 layers structured synthesis on top of the v0.5.8 Evidence Graph. It introduces depth profiles (Undergraduate, Masters, PhD) that deterministically change report structure, rigour, and reference density. Outputs include argument maps, agreement/disagreement tables, and a gaps & next-steps registry. All content remains attribution-faithful to anchored sources.

2) Scope

Goals

  • Depth profiles (UG/MSc/PhD) that alter: section set, granularity, target ref counts, critical appraisal intensity.
  • Argument mapping per research question with supports/contradicts links to claims and anchors.
  • Tables of agreement/disagreement and an explicit uncertainty & gaps registry.
  • Deterministic transformation from Evidence Graph → Synthesis → Report.
  • Externalised profile configs and outline templates (JSON).

Non‑Goals (now)

  • PRISMA screening UI and triage (v0.6.0).
  • Protocol/BOM generation (v0.6.1).
  • Human rater tooling (v0.5.10).

Key properties

  • Deterministic synthesis: fixed prompts + seeds + cache.
  • Attribution preserved: every assertion ties back to claim→source_anchor.
  • Repeatable structure per profile via external templates.

4) Components

4.1 Question Decomposer

  • Breaks topic into sub‑questions using rule‑based patterns + LLM with fixed prompts.
  • Ensures each sub‑question maps to a claim neighbourhood in the Evidence Graph.

4.2 Argument Mapper

  • For each sub‑question, gathers supports/contradicts edges and scores consensus (ratio of supporting edges weighted by anchor confidence and source quality).

4.3 Synthesis Engine

  • Applies a Profile Strategy (UG/MSc/PhD) to choose: narrative depth, number of studies to cite, required method critique, statistical details.
  • Produces structured blocks: Background, Current Evidence, Conflicts, Limitations, Open Questions, Next Steps.

4.4 Outline Renderer

  • Converts synthesis blocks into a profile‑specific outline using external templates with required sections and target lengths.

4.5 Report Generator + Citation Weaver

  • Same engine as v0.5.8; ensures inline references map to source_id#page anchors.

4.6 Telemetry/Run Manifests

  • Records counts: claims touched, sources cited, consensus scores, profile used.

6) Interfaces

6.1 CLI

  • eunice synthesise --run <run_id> --profile ug
  • eunice render --profile ug
  • eunice outline validate --template templates/ug.json

6.2 Internal FastAPI (optional/local)

POST /synthesis/run -> {run_id, profile} -> {synthesis_run_id}
GET /synthesis/{id} -> {blocks, metrics}
POST /outline/validate -> {template_json} -> {valid:true|false, errors:[]}

7) Algorithms & Heuristics

  • Consensus score: weighted mean of support vs contradict edges where weights = anchor confidence × source_quality (e.g., journal/preprint heuristic) × recency factor.
  • Depth control: per profile targets (e.g., UG: 10–20 sources, minimal stats; MSc: 20–40 with basic stats; PhD: 40+ with method critique and limitations).
  • Conflict surfacing: require explicit Conflicts section when contradict weight > τ (config).

9) Observability

  • Metrics: sources cited, blocks generated, consensus histogram, conflicts triggered, token usage.
  • Artefacts: synthesis.json, report.md, outline_applied.json.

10) Security & Licensing

  • No new network surfaces beyond v0.5.8; templates loaded from local repo.
  • Preserve provenance in all blocks; no PII stored.

11) Performance Targets

  • Synthesis over 100–200 candidate sources ≤ 3 min with warm cache.
  • Rendering deterministic across runs with same profile.

12) Testing & CI

  • Snapshot tests per profile (UG/MSc/PhD) ensuring required sections exist and length windows hold.
  • Attribution tests: every claim in blocks maps to valid source_id#page.
  • Consensus sanity: conflicting inputs produce expected Conflicts section.

13) Migration Plan

  • Add new tables (profile, outline_template, synthesis_run, synthesis_block, block_citation).
  • Seed default profiles + outlines.
  • Wire CLI and renderer to use profiles.

14) Risks & Mitigations

  • Over‑aggressive consensus → calibrate weights and expose in config.
  • Template drift → SHA256 pin + schema validation.
  • Brittle decompositions → fall back to rule‑based question sets for known topics.

15) Acceptance (DoD)

  • UG/MSc/PhD profiles produce measurably different structures and depth.
  • 100% of claims in synthesis have valid attributions.
  • Deterministic reruns produce identical synthesis.json and report.md.
