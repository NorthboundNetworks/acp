Detailed Description

Contract Tests - Golden Transcripts

This directory contains golden transcript files that document expected CLI behavior and telemetry outputs for the companion mediation, blackbox logging, and gimbal control features.

Structure

Golden transcripts provide deterministic, repeatable evidence of feature behavior. Each transcript captures:

  1. Input commands (user CLI commands)
  2. Expected output (system responses, status messages)
  3. Context (system state, feature flags, test conditions)

Test Categories

Audit Commands (audit_*.txt)

  • audit_show.txt - Basic audit log display
  • audit_show_recent.txt - Recent entries with –recent flag

Mission Commands (mission_*.txt)

  • mission_list.txt - List available mission logs
  • mission_replay_summary.txt - Replay mission with –summary
  • mission_export_verify.txt - Export and HMAC verification

HMAC Commands (hmac_*.txt)

  • hmac_set_show.txt - Set key and show label (no secret exposure)

Gimbal Commands (gimbal_*.txt)

  • gimbal_cli.txt - All gimbal CLI operations (mode/set/sweep/status)

Telemetry Tests (telemetry_*.txt)

  • telemetry_burst.txt - Rate limiting test (≥20 events/s → ≤2 msgs/s)

Performance Tests (perf_*.txt)

  • perf_snapshots.txt - CPU usage and loop timing measurements

Bench Tests (*_bench_notes.txt)

  • gimbal_bench_notes.txt - Hardware measurements for SC-006 validation

Usage

  1. Capture: Run CLI commands and capture both input and output
  2. Sanitize: Remove timestamps and session-specific data
  3. Validate: Ensure outputs are deterministic and reproducible
  4. Document: Add context comments explaining test conditions

Format Guidelines

# Test: <Brief Description>
# Date: YYYY-MM-DD
# Context: <System state, feature flags, etc.>
> command_input
Expected output line 1
Expected output line 2
> next_command
Next expected output
# Comments explain non-obvious behavior

Constitution Compliance

These transcripts fulfill the observability and testability requirements:

  • CLI Contracts: Commands produce stable, human-readable output
  • Rate Limiting: Telemetry summaries respect 2 msgs/sec cap
  • Audit Trail: Companion decisions are operator-visible within 1s
  • Performance: Evidence that features don't degrade control loops
