Country Code Consolidation - COMPLETED ✅

Summary

All country code mappings (ISO2, ISO3, and name mappings) are now stored in the database (data/sql_app.db) as described in docs/DATABASE_SCHEMA_DOCUMENTATION.md. The database is the single source of truth for all country code conversions throughout the SHIELD system. Configuration files (config/countries.json) are now only used for auxiliary metadata.

Files Modified

1. backend/geopolitics/cyberThreatCollector.py

BEFORE: Hardcoded country mapping dictionaries in methods

def _get_country_code_mapping(self):
return {
'US': 'USA', 'CN': 'CHN', 'RU': 'RUS', ... # 50+ hardcoded entries
}

AFTER: Dynamic queries from the database using centralized config functions

def _get_country_code_mapping(self):
return get_all_country_codes_mapping()
def _get_two_letter_code(self, three_letter_code: str) -> str:
return get_country_code_2_letter(three_letter_code)

2. backend/geopolitics/financeCollector.py

BEFORE: Hardcoded World Bank mapping dictionary

self.country_code_mapping = {
'AF': 'AFG', 'AL': 'ALB', 'DZ': 'DZA', ... # 200+ hardcoded entries
}

AFTER: Single call to database-backed function

def _convert_to_wb_country_code(self, iso_2_code: str) -> str:
return get_country_code_3_letter(iso_2_code)

BEFORE: Hardcoded country name to code mappings

this.countryMapping.set('ukraine', 'UA');
this.countryMapping.set('russia', 'RU');
// ... 30+ hardcoded entries

AFTER: Async loading from database via backend API
