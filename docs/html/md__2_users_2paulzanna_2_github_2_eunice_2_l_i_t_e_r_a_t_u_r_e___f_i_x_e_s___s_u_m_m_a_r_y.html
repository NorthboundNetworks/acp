Literature Worker Reference and Citation Extraction Fixes

Issues Identified and Resolved

1. Reference Extraction Issue (0 returned references)

Problem:

  • The code was incorrectly handling DOI formatting, adding "doi.org/" prefix before searching
  • The logic for extracting referenced_works from OpenAlex responses was flawed
  • Error handling was insufficient for malformed responses

Root Cause:

# BEFORE (problematic code):
if doc.get('doi').startswith("10."):
doc['doi'] = "doi.org/" + doc['doi'] # This corrupted the DOI for API calls
refers = await self.client.literature_search(doi, "openalex", 1, False)
# Logic assumed specific response structure that wasn't always present

Fix Applied:

  • Separated display formatting from API calls
  • Preserved original DOI format for OpenAlex API searches
  • Added robust handling for different OpenAlex response structures:
    • Direct response with referenced_works at top level
    • Response with results array containing document data
  • Added proper error handling and validation

2. Citation Extraction Issue (double URL problem)

Problem:

  • Code was passing complete cited_by_api_url to literature_search() function
  • The literature_search() function expected a query, not a complete URL
  • This resulted in malformed double URLs like: https://api.openalex.org/works/https://api.openalex.org/works?filter=cites:W4361213501?mailto=mail@paulzanna.com

Root Cause:

# BEFORE (problematic code):
cited_by_url = doc.get('cited_by_api_url') # This is a complete API URL
cites_response = await self.client.literature_search(cited_by_url, "openalex", 50, False)
# literature_search() treats cited_by_url as a query and builds another URL around it

Fix Applied:

  • Extract OpenAlex ID from the cited_by_api_url using regex pattern
  • Parse URL format: https://api.openalex.org/works?filter=cites:W4361213501
  • Extract just the ID (W4361213501) and create proper query (cites:W4361213501)
  • Pass the query format that literature_search() expects

3. DOI Format Management

Problem:

  • DOI formatting was applied too early, corrupting the DOI for API searches
  • Mixed display format with internal data format

Fix Applied:

  • Separate display formatting (for logging) from internal data storage
  • Only apply "doi.org/" prefix when displaying results or storing final output
  • Preserve clean DOI format for all API interactions
