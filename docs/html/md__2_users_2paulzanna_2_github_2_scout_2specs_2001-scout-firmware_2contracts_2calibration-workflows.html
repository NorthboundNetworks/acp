Calibration Workflows Contract

Created: 2025-10-25
Branch: 001-scout-firmware
Spec: ../spec.md

Scope

Define operator-facing calibration workflows and acceptance for sensors requiring calibration: IMU (basic level) and Magnetometer (full hard/soft iron routine). Expose via CLI with clear prompts and outcomes.

Common Rules

  • Safety: calibration is only permitted when disarmed; motors must remain disabled.
  • Mode: calibration runs in CONFIG mode; attempting in FLIGHT is rejected with error.
  • Prompts: human-readable step-by-step instructions; final RESULT=OK|FAIL with reason.
  • Persistence: successful calibration updates parameters and requires save to persist.
  • Evidence: status reflects calibration state; STATUSTEXT provides summary.

IMU Calibration (MPU-6050 class)

  • Command: cal imu
  • Steps:
    1. Prompt: "Place airframe stationary on a level surface. Press ENTER to start."
    2. Acquire bias samples over a short window (e.g., 3–5 s) with stability check.
    3. Compute gyro/accel biases; reject if motion detected beyond threshold.
    4. Show computed offsets and prompt to accept.
    5. Apply offsets; mark calibration state.
  • Acceptance:
    • Stationary variance below threshold during sampling window.
    • Offsets within expected bounds; write to config on save.
  • Errors:
    • 409 busy/armed; 400 motion detected; 500 sensor read failure.

Magnetometer Calibration (HMC5883L class)

  • Command: cal mag
  • Steps:
    1. Prompt: "Rotate airframe slowly in 3D to cover all orientations (60–90 s)."
    2. Collect samples; compute sphere fit for hard-iron offset and soft-iron matrix.
    3. Validate fit quality (residual error threshold).
    4. Show gains/offsets and prompt to accept.
    5. Apply calibration; mark calibration state.
  • Acceptance:
    • Fit residuals within threshold; no axis saturation flags during routine.
    • Heading changes smoothly post-calibration; health remains OK during slow rotation.
  • Errors:
    • 409 busy/armed; 400 insufficient coverage; 500 sensor read failure.

Operator Feedback & Telemetry

  • CLI prints step progress and final RESULT.
  • Telemetry uses STATUSTEXT for start, success, failure with reasons.
  • status includes calibration flags and last calibration timestamp.
